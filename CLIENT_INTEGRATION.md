## Инструкция по интеграции промокод-системы для клиента

### Базовый URL
```
https://ваш-домен.com/api
```

### Аутентификация
Все запросы к промо-эндпоинтам требуют заголовок:
```
X-API-Key: your-api-key-for-external-site
```
(Ключ выдаёт администратор, хранится в `.env` сервера как `API_KEY`)

---

### Эндпоинт 1: Проверка промокода

**Когда вызывать:** пользователь ввёл промокод и нажал "Применить"

```
POST /api/promo/check
Content-Type: application/json
X-API-Key: <ваш-ключ>

{
  "code": "TESTCODE",
  "externalUserId": "id-пользователя-в-вашей-системе"
}
```

**Ответы:**

| Ситуация | HTTP | Ответ |
|---|---|---|
| Код блогера | 200 | `{"type": "blogger", "bloggerName": "Имя блогера"}` |
| Премиум-код | 200 | `{"type": "premium", "durationDays": 7}` |
| Премиум уже использован | 410 | `{"error": "Промокод уже использован"}` |
| Код не найден | 404 | `{"error": "Промокод не найден"}` |

**Логика на клиенте:**
- Если `type === "blogger"` — показать "Промокод блогера {bloggerName} применён", запомнить код для отправки при покупке
- Если `type === "premium"` — активировать премиум на `durationDays` дней, промокод одноразовый и уже помечен использованным
- Если ошибка — показать сообщение из `error`

---

### Эндпоинт 2: Фиксация покупки

**Когда вызывать:** пользователь оплатил подписку и у него был применён промокод блогера

```
POST /api/promo/purchase
Content-Type: application/json
X-API-Key: <ваш-ключ>

{
  "code": "TESTCODE",
  "externalUserId": "id-пользователя-в-вашей-системе"
}
```

**Ответ:** `200 {"success": true}`

> Вызывать **только** для блогерских промокодов (type === "blogger"). Для премиум-кодов вызывать не нужно — они обрабатываются полностью в `/check`.

---

### Пример потока

```
1. Пользователь вводит код "SALE2026"
2. Клиент → POST /api/promo/check {code: "SALE2026", externalUserId: "usr_123"}
3. Сервер → {type: "blogger", bloggerName: "Иван"}
4. Клиент запоминает код, показывает "Промокод Ивана применён"
5. Пользователь покупает подписку, оплата проходит
6. Клиент → POST /api/promo/purchase {code: "SALE2026", externalUserId: "usr_123"}
7. Готово — в статистике блогера появится ввод + покупка
```
